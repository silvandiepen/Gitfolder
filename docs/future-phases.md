# Future Phases

## Phase 2: Better GitHub integration

- GitHub OAuth login.
- Pick repositories from a list.
- Create a new GitHub repository from the app.
- Better connected account status.
- HTTPS token-based Git access.

## Phase 3: Better folder management

- Per-folder ignore editor.
- Folder health checks.
- Large file warnings.
- Excluded file preview.
- Per-folder sync history.
- Manual snapshot button per folder.

## Phase 4: Restore and history

- View previous snapshots.
- Restore a file from history.
- Restore an entire folder snapshot.
- Compare current folder to previous snapshot.

## Phase 5: More providers

- GitLab.
- Bitbucket.
- Gitea.
- Self-hosted Git remotes.

## Phase 6: Product polish

- Proper onboarding.
- Launch at login.
- Notification preferences.
- App icon.
- Signed/notarized builds.
- App Store readiness if desired.

## Possible positioning later

GitFolder can remain a developer utility, or it can become a simpler product for non-developers.

Developer positioning:

```txt
Automatic Git snapshots for local folders.
```

Normal-user positioning:

```txt
Versioned folders, without touching Git.
```
